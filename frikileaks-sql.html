SQL Del Latín LQL y a su vez del Griego que significa LOL ha deribado en una sigla yanky que significa Sá Quedao Lelo, que traducido al español viene a significar "lenguaje de interrogatorio".

Nota: No es un término policial, como cabría pensar, sino informático.

Creado por Enrique Archi y Gomaespuma en el siglo XII durante la primera cruzada con el fin de poder enumerar a las tropas cristianas. En la actualidad se utiliza para torturar a los alumnos. El conocimiento de este lenguaje es imprescindible para liarla parda en una base de datos, o incluso cepillarse un montón de información valiosa. Se dice que algunos bichos raros pueden Mutilizar el SQL para mantener una base de datos operativa, coherente, actualizada e incluso optimizada, prueba de ello es el ilustre Enrique Archi, profesor de base de datos en Antequera y principal creador. Hay quien dice que el mito procede de los delfines. Esta teoría no es valida dado los últimos descubrimiento en el Pedro Espinosa (Antequera).

Sentencias más comunes

  • INSERT: Para introducir líneas duplicadas o ficticias (algún cabrón ideó una putada llamada unique index para impedir las líneas duplicadas, pero afortunadamente ni Dios lo usa como es debido).
  • UPDATE: Para cambiar indiscriminadamente valores correctos a erróneos en líneas previamente existentes.
  • DELETE: La favorita. Poderosa herramienta para cepillarse a saco un montón de información valiosísima e irrecuperable.
  • DROP: El arma definitiva. Rara vez se usa salvo en situaciones límite, porque aunque es muy destructiva suele dañar también al atacante.
